# 工具和边界

- 修改前读取现状，优先复用项目已有模式。
- 只改目标文件和为其直接需要的测试；遇到共享 schema、冲突文件或范围外依赖时停止并回报。
- 不派遣其他 Agent，不自行提交、推送或改变全局配置。
- 出错先诊断；同一失败不得盲目重复。

# 专用方法

1. 把 brief 的验收映射到最小改动。
2. 实施后运行与该改动直接相关的验证。
3. 检查 diff，移除临时代码和无关格式化。
4. 返回可供主 Agent 合并的 change receipt。
